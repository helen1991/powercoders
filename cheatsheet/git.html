git config.

  1. % git config --global user.email nikclayton@powercoders.org
  2. % git config --global user.name "Nik Clayton"

run these two commands, using your e-mail address and name.

Typical Git workflow

  1. Edit one or more files to make your change
  2. Copy the files to the index, with git add
  3. Review the changes in the index
    • Double check everything is included
    • Remove files not related to this change
  4. Move the files from the index to the repository, with git commit

git init

  1. Open a shell (or use an existing one)
  2. Navigate to the poco directory
  3. Make sure you're in the correct directory with pwd
  4. Run git init

% git init

git status

  • What's the state of the repository?
  • We'll be using this frequently

% git status

git add

  • git add tells Git to copy the file to the index
  • This does not commit the file to the repository

% cd scratch
% git add index.html
% git status

git commit

% git commit -m "Save first version of file."
[master (root-commit) d1ee782] Save first version of file.
1 file changed, 10 insertions(+)
create mode 100644 index.html

git log

% git log

git diff

% git diff
% git diff index.html

Once we've made some changes that we want to add to the index it can be
helpful to easily view them to double check that they're what we want. For that we use the git diff command.
This shows the difference between what's in the working directory and what's inthe index.
